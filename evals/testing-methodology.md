# 三层思维 Agent 测试方法论

基于 Superpowers `writing-skills` 的 RED-GREEN-REFACTOR 方法论，适配为 Skill 测试流程。

---

## 测试流程

### RED：Baseline（不带 Skill）

**目的**：记录 Claude 在没有 skill 时的默认行为，建立对照基线。

**方法**：
1. 在不加载 skill 的会话中，直接用相同输入 prompt 运行场景
2. 记录输出字数、结构、是否遗漏关键分析要素
3. 保存为 `evals/iteration-N/baseline-testX.md`

**关键指标**：
- 输出字数（中文字符数，用 `[一-鿿]` regex 匹配）
- 是否包含核心分析要素（数据可信度标注、一致性检查等）
- 是否存在已知偏差（恐惧式结论、技术自嗨等）

### GREEN：With-Skill（带 Skill）

**目的**：验证 skill 是否有效解决了 baseline 中发现的问题。

**方法**：
1. 在加载 skill 的会话中，用完全相同的输入 prompt 运行场景
2. 逐项检查 skill 中定义的所有规则是否被遵守
3. 与 baseline 对比，验证简洁性约束是否满足

**检查清单**：
- [ ] 字数约束：摘要模式 Step 1-4 ≤ 1200字, Step 5 ≤ 600字
- [ ] 字数约束：完整模式 Step 1-4 ≤ 3000字, Step 5 ≤ 1500字
- [ ] 无角色切换声明（摘要模式）
- [ ] 无对齐声明段落（摘要模式）
- [ ] 因果映射简化（摘要模式）
- [ ] 数据可信度标注（📊/🧮/💡）
- [ ] 禁止恐惧式结论
- [ ] 一致性检查存在
- [ ] Gate 函数生效（信息不足时阻断）

### REFACTOR：边界场景（压力测试）

**目的**：找到 skill 在边界条件下的漏洞，迭代修复。

**方法**：构造压力场景，验证 skill 鲁棒性。

## 字数统计方法

```python
import re
chinese_chars = len(re.findall(r'[一-鿿]', text))
```

只统计中文字符，不含标点、英文、数字。这是评估简洁性的核心指标。

## 迭代节奏

```
v0.1.0 → 添加核心框架 → RED baseline → GREEN 验证 → 发现问题
v0.2.0 → 修复简洁性问题 → GREEN 验证 → 压力测试 → 发现新问题
v0.3.0 → 修复 brief 模式问题 → GREEN 验证 → 全量测试通过
v0.4.0 → Superpowers 优化（CSO/token/anti-bias/gate）→ 待验证
```

每次迭代遵循：
1. 明确本轮要解决的具体问题
2. 修改 SKILL.md
3. 用已有测试场景回归验证（确保不退化）
4. 用新压力场景测试新功能
5. 更新 final-summary.md
